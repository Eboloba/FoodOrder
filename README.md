## FoodOrder

# FoodOrder - Сервис доставки еды

Веб-приложение для заказа еды из ресторанов с функциями авторизации, корзины, оформления заказов и администрирования.

## Описание проекта

FoodOrder - это полнофункциональный сервис доставки еды, который позволяет пользователям:
- Регистрироваться и авторизовываться в системе
- Просматривать каталог блюд по категориям
- Искать блюда по названию
- Добавлять товары в корзину
- Оформлять заказы с выбором адреса доставки
- Просматривать историю своих заказов
- Управлять профилем (изменять данные, менять пароль)
- Для администраторов: управлять каталогом блюд и заказами

## Функционал

### Для пользователей:
- **Регистрация** - создание нового аккаунта с валидацией данных
- **Авторизация** - вход в систему по email и паролю
- **Восстановление пароля** - через email
- **Каталог блюд** - просмотр блюд по категориям (Пицца, Суши, Бургеры, Салаты, Десерты, Напитки)
- **Поиск** - поиск блюд по названию
- **Сортировка** - по цене, рейтингу, времени доставки
- **Корзина** - добавление/удаление товаров, изменение количества
- **Оформление заказа** - выбор адреса доставки
- **История заказов** - просмотр всех своих заказов
- **Профиль** - редактирование личных данных и смена пароля

### Для администраторов:
- **Управление блюдами** - добавление, редактирование, удаление блюд
- **Управление заказами** - просмотр всех заказов, изменение статуса
- **Категории** - управление категориями блюд
- **Рестораны** - управление списком ресторанов

## Технологический стек

### Frontend:
- **HTML5** - разметка страниц
- **CSS3** - стилизация интерфейса
- **JavaScript (Vanilla)** - клиентская логика
- **Font Awesome 6.0** - иконки

### Backend:
- **ASP.NET Core Web API** 
- REST API для взаимодействия с frontend

### API Endpoints:
```
Authentication:
- POST /api/Auth_Regis/Auth - Авторизация
- POST /api/Auth_Regis/Regis - Регистрация
- POST /api/Auth_Regis/ResetPassword - Восстановление пароля
- POST /api/Auth_Regis/ChangePasswordProfile - Смена пароля

Products:
- GET /api/ProductsView/GetProduct - Получить список блюд
- GET /api/ProductsView/GetProduct/{id} - Получить блюдо по ID
- POST /api/ProductsView/AddDish - Добавить блюдо
- POST /api/ProductsView/EditDish/{id} - Редактировать блюдо
- DELETE /api/ProductsView/DeleteDish/{id} - Удалить блюдо
- GET /api/ProductsView/GetAllCategories - Получить все категории
- GET /api/ProductsView/GetAllRestaurants - Получить все рестораны

Cart:
- GET /api/Cart/GetCartItems - Получить содержимое корзины
- POST /api/Cart/AddItem - Добавить товар в корзину
- POST /api/Cart/UpdateCartItem - Обновить количество товара
- POST /api/Cart/PlaceOrder - Оформить заказ
- GET /api/Cart/GetCartItemCount - Получить количество товаров в корзине

Orders:
- GET /api/Orders/GetUserOrders - Получить заказы пользователя
- GET /api/ProductsView/GetAllOrders - Получить все заказы (админ)
- POST /api/ProductsView/UpdateOrderStatus/{id} - Изменить статус заказа (админ)

Profile:
- POST /api/Profile/UpdateProfile - Обновить профиль пользователя
```

## Установка и запуск

### Требования:
- Веб-сервер для хостинга frontend (можно использовать Live Server в VS Code)
- Backend ASP.NET Core приложение на порту 5233

### Запуск:
1. Убедитесь, что backend API запущен на `http://localhost:5233`
2. Откройте `main.html` в браузере через веб-сервер
3. Зарегистрируйтесь или войдите в систему
4. Начните пользоваться приложением

### Доступы:
- **Пользователь:** roleId <= 1 (обычный пользователь)
- **Администратор:** roleId > 1 (доступ к admin.html)

## Безопасность

- Хеширование паролей на стороне сервера
- Валидация данных на клиенте и сервере
- Защита от XSS через безопасное использование localStorage
- Кодирование данных пользователя в заголовках запросов (Base64)

## Статусы заказов

- **Новый** - заказ только что оформлен
- **В работе** - заказ принят и готовится
- **Выполнен** - заказ доставлен
- **Отменён** - заказ отменён

## Известные ограничения

- Все данные хранятся на сервере в базе данных
- Изображения блюд должны быть загружены вручную в папку `images/food/`
- Адреса доставки ограничены предустановленным списком
